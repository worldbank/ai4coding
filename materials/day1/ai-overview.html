How LLMs Work

Generative AI and large language models (LLMs) are probabilistic: they predict the most probable sequence of words that could follow a given prompt, as if a human had written both the prompt and the response.

Because predictions are probabilistic, a single prompt can yield an infinite number of completions. Your goal as a user is to make the AI’s responses as deterministic and accurate as possible. The more precise your prompt and the richer the context you provide, the more useful the answer.

Three skills determine how well you can do this:

  1. Writing effective prompts — clear, specific, and structured requests
  2. Curating context — giving the model exactly the information it needs, nothing more
  3. Choosing the right interface — inline completion, chat assistant, or agent mode (see below)

Strengths and Limitations

When AI assistance helps

  • Boilerplate and repetitive code patterns
  • Translating code between languages (Stata → R, R → Python)
  • Explaining unfamiliar code or error messages
  • First drafts of documentation, comments, and reproducibility packages
  • Searching for the right function or package across a large API

When AI assistance is unreliable

  • Domain-specific logic that requires knowledge of your data
  • Long multi-step analyses where errors compound silently
  • Any task where you cannot verify the output
  • Up-to-date information (models have a knowledge cutoff)
  • Numerical results — always verify against your own calculations
Warning

Never trust AI output blindly. Treat AI-generated code as a first draft that must be reviewed, tested, and understood before use.

Responsible Use

Sensitive data

Do not paste sensitive, confidential, or personally identifiable data into any AI prompt or chat window. This includes microdata, individual-level survey responses, restricted datasets, or anything covered by a data use agreement.

Best practice: work with synthetic or anonymised examples when asking AI for help with data manipulation.

Limiting context

Sharing unnecessary files or long codebases with an AI agent increases cost, slows responses, and can expose data unintentionally. Give the model only the context directly relevant to the task at hand.

Setting the right expectations

  • AI is an assistant, not a replacement for analytical judgment.
  • You remain responsible for the correctness and reproducibility of your work.
  • Errors introduced by AI are your errors.

Effective Prompting

A good prompt contains four elements:

Element Purpose Example
Role Frame the model’s perspective “You are an expert Stata programmer.”
Task What you want done “Refactor the following do-file to remove hard-coded paths.”
Context Relevant background “This code uses household survey data. The file paths are in a local macro root.”
Format How to return the answer “Return only the revised do-file, with a one-line comment above each changed block.”

Prompting tips

  • Be specific: “replace the loop with egen” beats “make it faster”
  • Break complex tasks into steps; ask for one step at a time
  • If the first answer is wrong, correct it explicitly rather than starting over
  • Ask the model to explain its changes so you can learn from them

Curating Context

AI agents read the files and variables you show them. The quality of the answer depends directly on the quality of the context:

  • Open only the files that are relevant to the task
  • Close unrelated tabs and panels before starting an agent task
  • Use prompt files (.prompt.md) and skills to provide standing instructions without re-typing them every session

Prompt files and skills are covered in detail in Day 2 → Advanced Topics.

Stata vs R vs Python

AI support quality varies across languages:

Language Copilot support Notes
R ✅ Excellent Largest open-source code corpus; excellent tidyverse coverage
Python ✅ Excellent Best-supported language overall
Stata ⚠️ Good, with caveats Smaller training corpus; Stata MCP extension significantly improves agent performance; always verify output
Note

Stata support has improved significantly with the Stata MCP extension, which gives the AI agent live access to your Stata environment. See Positron Extensions for setup.

Three Modes of AI Assistance

Mode What it does When to use
Inline completion Suggests the next line or block as you type Always — an instant productivity boost
Chat / Assistant Answers questions with awareness of your open files Explaining code, finding functions, quick fixes
Agent Plans and executes multi-step tasks autonomously Refactoring, reproducibility packages, complex analysis pipelines

Hands-on practice with all three modes is in Case Studies.
